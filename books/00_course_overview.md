# 课程总览

本课程用 8 周、16 课时讲解现代程序设计的基础工作流：先掌握 Bash 与 Git，再学习如何把 AI 辅助编程（Vibe Coding）纳入工程过程，最后构建能够稳定产出 specification 的编写机器。

## 学习目标

- 能在命令行中高效完成文件、文本、进程和项目管理任务。
- 能用 Git 组织个人开发历史，并处理基础协作场景。
- 能把 AI 工具用于理解、实现、调试、重构和测试，而不是只生成代码片段。
- 能编写清晰、可验证、可迭代的 specification，并把它连接到代码实现和验收测试。

## 课程安排

| 周次 | 课时 | 主题 | 课堂产出 |
| --- | --- | --- | --- |
| 第 1 周 | 1 | Bash 与命令行心智模型 | 能解释路径、工作目录、标准输入输出 |
| 第 1 周 | 2 | 文件系统、通配符与文本查看 | 完成一次课程资料整理脚本 |
| 第 2 周 | 3 | 管道、重定向与文本处理 | 用管道完成日志/CSV 的轻量分析 |
| 第 2 周 | 4 | Bash 脚本与自动化 | 写出带参数、错误处理和帮助信息的小脚本 |
| 第 3 周 | 5 | Git 基础对象模型与提交 | 建立可读的提交历史 |
| 第 3 周 | 6 | 分支、合并与冲突 | 完成一次冲突定位和修复 |
| 第 4 周 | 7 | Git 远程协作工作流 | 使用 pull request 或等价流程协作 |
| 第 4 周 | 8 | 代码审查与变更说明 | 写出清晰的变更摘要和 review 反馈 |
| 第 5 周 | 9 | Vibe Coding 的工作边界 | 设计一次 AI 辅助实现任务 |
| 第 5 周 | 10 | 用 AI 读代码、定位问题与调试 | 将错误报告转化为可验证修复 |
| 第 6 周 | 11 | AI 辅助重构与测试补齐 | 在保持行为不变的前提下重构代码 |
| 第 6 周 | 12 | 从 prompt 到工程循环 | 建立“需求-实现-运行-验证-修正”的循环 |
| 第 7 周 | 13 | Specification 的结构 | 写出目标、约束、输入输出和验收标准 |
| 第 7 周 | 14 | 可执行规格与测试设计 | 将 specification 转化为测试用例 |
| 第 8 周 | 15 | Specification 编写机器 | 设计一套从需求到规格的模板化流程 |
| 第 8 周 | 16 | 综合项目展示与复盘 | 提交一个从规格到实现再到验证的小项目 |

## 课程成果

课程结束时，学生应能独立完成一个小型软件任务：先写 specification，再用 Git 管理开发过程，借助 AI 工具完成实现和调试，最后用可运行示例或测试验证结果。
